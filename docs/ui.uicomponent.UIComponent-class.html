__init__(self, parent, surface, graphics, surface_params, graphics_params)
(Constructor)

source code 

The constructor for UIComponent

Parameters:
  • parent (ui.uicomponent.UIComponent) - The parent component which contains this component. This should be an instance of UIComponent or any of its sub-classes. All components should have a parent with the exception of ui.screen.Screen. If you wish to draw a component directly to the screen, then its parent should be set to ui.screen.Screen. If a component is going to be added to a UIContainer using either ui.uicomponent.UIContainer.addComponent or ui.uicomponent.UIContainer.addComponents, then parent may be set to None, as either of those methods will set the parent attribute of the added components.
  • surface (type) - The type of display surface to use when displaying this component. This should be ui.displaysurface.DisplaySurface or one of its subclasses.
  • graphics (type) - The type of graphics object to control how the component displays. This should be a subclass of ui.uigraphics.UIGraphics.
  • surface_params (ui.parameters.Parameters) - The parameters that will be used to instantiate the display surface for this component. This should be a subclass of ui.parameters.Parameters.
  • graphics_params (ui.parameters.Parameters) - The parameters that will be used to instantiate the graphics object for this component. This should be a subclass of ui.parameters.Parameters.
Post Conditions:
  • This component's display_surface attribute references an instance of type surface initialized with surface_params.
  • This component's graphics attribute references an instance of type graphics initialized with graphics_params.

Instance Variable Details [hide private]

visible

A boolean representing whether or not the component is currently visible.

Values:

  • True if the component is currently visible
  • False otherwise.

The visibility of a component controls whether the component is rendered, as well as whether it listens for events.

state

A string representing the state of this component with respect to the mouse.

Possible states are:

  • normal: The default state. This means that the mouse is not currently interacting with this component
  • hover: The mouse is currently hovering over this component.
  • pressed: The left mouse button has been pressed on this component.

Note: The state will always be normal unless this component is listening to ui.internals.MOUSEMOTIONEVENT, ui.internals.MOUSEBUTTONDOWNEVENT, or ui.internals.MOUSEBUTTONUPEVENT

See: setState, onMouseMove, onMouseOnePress, onMouseOneRelease, mouseMoveCollide, mouseMoveMiss, pressedMouseMoveCollide, pressedMouseMoveMiss, mouseOnePressCollide, mouseOnePressMiss, mouseOneReleaseCollide, mouseOneReleaseMiss, pressedMouseOneReleaseCollide, pressedMouseOneReleaseMiss

events

A dictionary containing the event types that this component is currently listening to.

  • key: the ui_type of the event
  • val: the function to be called in response to the event

See: registerEvent, addToDispatcher, addToDispatchers, removeFromDispatcher, removeFromDispatchers

timers

A dictionary containing the event timers owned by this component.

  • key: the timer object
  • val: True

The values for all keys are always True, and hold no particular meaning.

See: registerTimer, addTimer, addTimers, removeTimer, removeTimers
